Web Services

Web Services dienen in erster Linie dazu, Anwendungssysteme unterschiedlicher Technologien miteinander zu verbinden. So ist es z. B. möglich, dass ein in C# geschriebener Windows Client über Web Services mit einem Java EE Server kommuniziert. Web Services sind unabhängig von der verwendeten Programmiersprache, da Sie auf XML basieren.

Klassischerweise werden Web Services mit Hilfe des SOAP Protokolls übertragen. Eine schlankere und performantere Methode ist allerdings die Verwendung von RestFul-Webservices.

Näheres zu Verwendung von Rest im Java Bereich beschreibe ich im Artikel RESTFul Web Services und in meinem Blog.
